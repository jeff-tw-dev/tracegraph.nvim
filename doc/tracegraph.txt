*tracegraph.txt*  Recursive call-trace tree based on LSP callHierarchy

==============================================================================
INTRODUCTION                                                      *tracegraph*

tracegraph.nvim opens a side panel that expands caller-of-caller (incoming)
or callee-of-callee (outgoing) chains level by level, using the LSP
callHierarchy protocol. Each expand issues one LSP request, so the tree is
lazy by construction.

Requires Neovim 0.10+ and an LSP server supporting
textDocument/prepareCallHierarchy.

==============================================================================
API                                                           *tracegraph-api*

require("tracegraph").setup({opts})                       *tracegraph.setup()*
    Optional. Merge {opts} into the defaults (see |tracegraph-config|).

require("tracegraph").open({direction})                    *tracegraph.open()*
    Open the panel for the function under the cursor.
    {direction} is "incoming" (default) or "outgoing".

require("tracegraph").close()                             *tracegraph.close()*
    Close the panel.

:Tracegraph [incoming|outgoing]                                  *:Tracegraph*
    Command form of open(). Defaults to incoming.

==============================================================================
PANEL KEYS                                                   *tracegraph-keys*

Defaults (configurable via setup):

    o / <Tab>   expand/collapse node (lazy LSP request)
    <CR>        jump to call site
    gd          jump to definition
    p           preview call site in a float
    s           switch incoming/outgoing
    q           close panel

==============================================================================
CONFIGURATION                                              *tracegraph-config*

setup() is optional; pass only what you want to change (deep-merged).
Defaults: >lua
    require("tracegraph").setup({
      panel = {
        width = 56,
        position = "right",
      },
      preview = {
        context = 7,
        max_width = 100,
        border = "rounded",
      },
      keys = {
        expand = { "o", "<Tab>" },
        jump = "<CR>",
        definition = "gd",
        preview = "p",
        switch = "s",
        close = "q",
      },
      icons = {
        loading = "…",
        recursive = "↻",
        leaf = "·",
        expanded = "",
        collapsed = "",
      },
    })
<
                                                     *tracegraph-config-panel*
panel.width         (number, default 56)
    Width of the tree panel in columns. The panel sets 'winfixwidth', so
    it keeps this width across layout changes.

panel.position      (string, default "right")
    "right" opens the panel as a botright vsplit, "left" as a topleft
    vsplit.

                                                   *tracegraph-config-preview*
preview.context     (number, default 7)
    Lines of source shown above and below the call site in the preview
    float. The float's height is 2 * context + 1, clamped at file
    boundaries.

preview.max_width   (number, default 100)
    The float's width follows the editor width minus margins, but never
    exceeds this.

preview.border      (default "rounded")
    Passed straight to |nvim_open_win()|: "single", "double", "none", a
    char array, ...

                                                      *tracegraph-config-keys*
keys.*
    Panel-local keymaps. Each value is a single key ("q") or a list of
    keys ({ "q", "<Esc>" }); every listed key is mapped. The first key of
    each entry is shown in the panel's help header.

    expand      expand/collapse node; first expand of a node issues one
                LSP request
    jump        jump to the call site in the previous window (the root
                has no call site, so it jumps to the definition)
    definition  jump to the node's definition
    preview     open the call-site float; closes on cursor move
    switch      reopen the same root in the opposite direction
    close       close the panel

                                                     *tracegraph-config-icons*
icons.*
    Node state markers: loading (request in flight), recursive (node
    already in its own ancestor chain), leaf (no further callers or
    callees), expanded, collapsed.
